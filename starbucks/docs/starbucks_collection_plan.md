# 스타벅스 전국 매장 정보 수집 계획서 (Starbucks Store Data Collection Plan)

본 계획서는 스타벅스 코리아 공식 홈페이지의 매장 찾기 서비스에서 내부적으로 사용하는 API를 활용하여, 전국 스타벅스 매장 정보를 안전하고 효율적으로 수집하기 위한 설계를 정의합니다.

---

## 1. 수집 개요 및 목적
- **목적**: 전국 스타벅스 매장의 지점명, 주소, 전화번호, 위경도 좌표, 매장 타입 등 상세 정보를 수집하여 데이터 분석용 기초 자료를 확보합니다.
- **수집 대상 URL**: `https://www.starbucks.co.kr/store/store_map.do`
- **수집 방식**: 웹 브라우저를 모방한 API 직접 호출 (`POST` 방식)
- **최종 출력**: `starbucks/data/starbucks_stores.csv`

---

## 2. API 명세 및 파라미터 구조

스타벅스 홈페이지에서 내부적으로 호출하는 다음 비공식 API 엔드포인트를 활용합니다.

### A. 시도 목록 조회 API
- **URL**: `https://www.starbucks.co.kr/store/getSidoList.do`
- **Method**: `POST`
- **Parameters**: 없음 (혹은 빈 페이로드)
- **Response**: JSON 형식
  - 주요 필드: `sido_cd` (시도 코드), `sido_nm` (시도 이름)

### B. 매장 정보 조회 API
- **URL**: `https://www.starbucks.co.kr/store/getStore.do`
- **Method**: `POST`
- **Parameters**:
  - `p_sido_cd`: 시도 코드 (예: 서울 `01`, 경기 `08`)
  - `p_gugun_cd`: 구군 코드 (시도별 전체 수집 시 빈 값 `""`)
  - `ins_lat`: 기준 위도 (예: `'37.56682'`)
  - `ins_lng`: 기준 경도 (예: `'126.97865'`)
  - `iend`: 한 번에 가져올 최대 매장 수 (안전하게 2000 정도로 설정)
  - `in_biz_cd`, `set_date`: 빈 값 `""`
- **Response**: JSON 형식
  - 주요 필드:
    - `s_name`: 매장명
    - `doro_address`: 도로명 주소
    - `tel`: 전화번호
    - `lat`: 위도
    - `lng`: 경도
    - `sido_name`: 시도 이름
    - `gugun_name`: 구군 이름
    - `store_area_code`: 매장 구분 코드

---

## 3. 수집 파이프라인 워크플로우

1. **공통 가상환경 활성화**: 워크스페이스 루트의 공통 `.venv` 사용.
2. **시도 코드 동적 조회**: `getSidoList.do`에 POST 요청을 전송하여 전국 17개 시도 코드를 확보합니다.
3. **루프 순회 및 매장 조회**:
   - 획득한 시도 코드를 순회하며 `getStore.do`에 POST 요청을 보냅니다.
   - 각 시도별로 매장 정보를 수집하여 임시 리스트에 병합합니다.
4. **랜덤 딜레이 적용**:
   - 서버 부하 방지 및 차단 우회를 위해 요청 간에 `0.3초 ~ 0.8초` 사이의 랜덤 딜레이를 부여합니다.
5. **예외 처리 및 지수 백오프**:
   - HTTP 연결 에러, 타임아웃, 429(Too Many Requests), 503 오류 발생 시 최대 5회 재시도합니다.
   - 재시도 대기 시간은 `2초 -> 4초 -> 8초 -> 16초 -> 32초`로 지수적으로 증가합니다.
6. **데이터 정제 및 저장**:
   - 수집된 데이터를 Pandas DataFrame으로 변환합니다.
   - 중복된 매장이 있는 경우 제거합니다.
   - 출력 포맷: CSV
   - 인코딩: 한글 깨짐 방지를 위해 `utf-8-sig`를 사용합니다.

---

## 4. 데이터 저장 스키마 (CSV)

수집된 데이터는 아래의 주요 컬럼들을 포함합니다.

| 컬럼명 (영문) | 컬럼 설명 (국문) | 데이터 예시 |
| :--- | :--- | :--- |
| `s_name` | 매장명 | 역삼역점 |
| `sido_code` | 시도 코드 | 01 |
| `sido_name` | 시도명 | 서울 |
| `gugun_code` | 구군 코드 | 0101 |
| `gugun_name` | 구군명 | 강남구 |
| `doro_address`| 도로명 주소 | 서울특별시 강남구 테헤란로 145 |
| `tel` | 전화번호 | 1522-3232 |
| `lat` | 위도 | 37.50062 |
| `lng` | 경도 | 127.03621 |
| `store_type` | 매장 타입 (DT, 일반 등) | 일반 |

---

## 5. 파이프라인 수행 규칙 준수 사항
- **파이썬 인코딩**: 파일 최상단 `# -*- coding: utf-8 -*-` 명시.
- **주석**: 모든 소스코드 주석 및 설명은 한국어로 상세 작성.
- **환경**: 워크스페이스 공통 `.venv` 기반 실행 (`uv run`).
- **폴더 규격**: 수집 스크립트는 `starbucks/src/`에 배치하고 결과물은 `starbucks/data/`에 적재.
