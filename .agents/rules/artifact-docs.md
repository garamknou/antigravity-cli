---

- **아티팩트 로컬 동기화 원칙**:
  - 에이전트가 글로벌 아티팩트 디렉터리(`<appDataDir>\brain\<conversation-id>\`)에 작성하는 모든 아티팩트 파일(예: `*_plan.md`, `walkthrough.md` 등)은, 진행 중인 프로젝트 폴더의 `docs/` 디렉터리 내에도 동일한 파일명으로 즉시 생성 또는 복사되어야 합니다.
  
- **프로젝트 독립적 관리**:
  - 시스템 전역 아티팩트가 제공하는 인터랙티브 승인(Proceed) 및 UI 표시 기능은 유지하되, 전체 히스토리와 최종 문서(계획서, Walkthrough, 리포트 등)는 프로젝트 내부 `docs/` 폴더 아래에서 통합 조회될 수 있도록 합니다.

- **대상 파일 포맷**:
  - 모든 마크다운 문서(`.md`) 아티팩트 및 기타 사용자 제공 지침 문서가 대상이 됩니다.
  
- **수동 복사 동기화**:
  - 만약 도구 제약으로 인해 실시간 동기화가 불가능한 경우, 에이전트는 작업을 마무리하기 전에 직접 수동 복사 명령어 혹은 도구 호출을 수행하여 로컬 폴더에 최신 상태를 반영해야 합니다.
